Explore the Project

An interactable Power BI dashboard was developed for TMT, a steel long product, with the objective of enabling easier exploration and analysis of sales data. The dashboard was designed to be user-friendly, with filters allowing users to dynamically adjust the view based on their specific needs. It displays sales value and features various graphs and visuals that help in sales analysis and identification of insights.

The initial version of the dashboard was created using data from a single state and a shorter time range, serving as a prototype to test the data structure, filtering functionality and visualization layout. This phase allowed for iterative refinements and validation of the dashboardâ€™s logic. Once the functionality and presentation were confirmed to be effective, the dashboard was scaled up to include all states and a complete time period, offering a comprehensive and organization-wide view of TMT sales performance.
This was developed in Python using Visual Studio Code and focused on building a machine learning-based forecasting model using the TMT sales data. The model was designed to address two key use cases: firstly, to predict sales for the next 12 months and secondly, to perform target-based prediction, where the user can either input a desired sales target to estimate when it will be achieved, or allow the model to suggest a target based on historical trends.

To implement this, the Prophet model developed by Meta Open Source formerly Facebook, was used for time series forecasting. It is well-suited for handling data with strong seasonality and irregularities such as missing values or outliers. The modelâ€™s outputs were visualized using Pythonâ€™s visualization libraries, making the predictions easy to interpret and valuable for planning and decision-making.
The chatbot was developed using HTML to solve a key challenge faced by internal users i.e. retrieving specific sales information quickly from a large Excel dataset. Manually searching through thousands of rows to locate sales figures for a particular customer, region, or time period was both time-consuming and prone to errors. The chatbot provides a streamlined solution by offering fast, accurate access to frequently required sales insights. A total of 27 predefined queries were implemented, based on real-world use cases identified during discussions with the Sales and IT Digital Solutions team.

These include queries such as top or bottom customers by sales, sales in a specific state or city, customer-wise quantity sold in a particular month and comparative analysis between periods or regions. To ensure ease of use, the chatbot interface was designed with clickable option for each query, rather than requiring users to type text inputs. This reduces the risk of input errors and simplifies the interaction for users. On selecting a query, the chatbot processes the request and returns an accurate response derived directly from the dataset. This approach significantly reduces the need for manual filtering and improves the speed and efficiency of internal reporting tasks.
